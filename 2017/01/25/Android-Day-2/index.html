Log

  • 너무 많은 로그는 표시가 안될 수 있음 (버퍼 용량 문제)
  1. ERROR : 에러 상황을 표시합니다
  2. WARN : 경고 상황을 표시합니다
  3. INFO : INFO 현재 각 단계에 맞는 상황을 표시합니다
  4. DEBUG : 개발 할 때는 보이지만 배포 때는 제외됩니다
  5. VERBOSE : 개발 할 때는 포함되지만 배포 할때는 컴파일이 되어서는
    안 됩니다

Thread

  1. Main Thread (aka UI Thread) - All user input + output
    • onPreExecute()
    • onProgressUpdate()
    • onPostExecute()
  2. Background Thread - for long-running work
    • doInBackground()
      진행 사항 보기
  • doInBackground를 진행하면서 publishProgress()를 이용하면 현재
    작업이 얼마나 진행되었는지 알 수 있습니다

NetworkOnMainThreadException

  • 네트워크에서 요청하는 코드는 long-running work이기 때문에 Background Thread로 처리하여야 한다.
